Pos Network Printer

Print POS orders and receipts by using Network printers

Key features:
  • it is enough to have one posbox to use any number of network printers
  • you just need to know IP address of network printer
  • it works for any model of network printers, which support esc/pos protocol
  • ability to print both order and payment receipts
  • ability to see connection status of each printer
  • ability to disable USB printers if needed

By default, odoo supports USB printers only for POS order printing. Actually, it is not flexible and there are some inconveniences:
  • if you have several USB printers and send receipt to print, it will be printed on the first one encountered, i.e. we have no chance to choose certain USB printer
  • you can use limited number of USB printer models only
  • you are not able to see connection status of each USB printer
The module eliminates these disadvantages, because makes it possible to print POS orders and receipts on Network printers.

How It Works

Go to Point of Sale → Configuration → Order Printers menu and open order printer form

  • Specify "Printer Name"
  • Specify IP address of network printer
  • Check the "Network Printer" box
  • Select product categories and click on "Save"

Go to Point of Sale → Configuration → Point of Sale and open POS config form.

  • Check the "Network Printer" box at Hardware Proxy/PosBox section
  • Optional: Specify "Network Receipt Printer IP" to be able to print POS receipts
  • Add network printer(s) to "Order Printers"


Open POS interface and add products. Click on the "Order" button to print order or make payment validation to print the receipt.

As a result you can see order/receipt printed on network printer.

Connection Status

Click on the PosBox icon at POS interface to check connection status of Network Printers.


You will be able to see common status for USB printers too (online/offline) if you use the latter ones

Note

It should be noted that the module requires Hardware Network Printer to be installed on PosBox. See also the "Documentation" tab for installation and configuration details.
